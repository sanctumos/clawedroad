## Contributing to Documentation

### How to Contribute

1. **Identify Issue**:
   - Outdated information
   - Missing information
   - Unclear explanation
   - Broken code example

2. **Make Changes**:
   - Update relevant file(s)
   - Test code examples
   - Update version and date
   - Update this changelog

3. **Submit**:
   - Create pull request
   - Describe changes
   - Link to related issues

### Documentation Standards

- Use Markdown format
- Include code examples
- Add table of contents for long documents
- Use consistent formatting
- Keep examples up to date
- Include version and date at bottom

### Style Guide

- **Headings**: Use sentence case
- **Code**: Use fenced code blocks with language
- **Links**: Use relative links for internal docs
- **Lists**: Use `-` for unordered, numbers for ordered
- **Emphasis**: Use `**bold**` for important, `*italic*` for emphasis
- **Code inline**: Use backticks for `code`, filenames, commands
